# 2026-06-01 — A0 + A1 landed (mobile-heap sequence)

Branch `mobile-heap-a0a1` off `846d0a8`. A0 and A1 are mutually defining: A0 says
what the system is; A1 is the executable check that A0's invariant holds on
multi-node topology. Neither stands alone.

## A0 — Target architecture + coherence invariant (spec)

Added normative **VTN0** to `spec/protocol/v2-turn-network.md`, ahead of VTN1:
- States the target: gossip-routed actor-local **mobile object heap** (the five
  load-bearing claims).
- Defines the **coherence invariant (CI):** for every durable cell there is
  exactly one authority (the commit-scope head); every other materialization is a
  content-addressed, provenance-tagged, read-through projection; no cell has two
  independent write paths; a derived copy is never a write-authority source.
- Records that the shipped DO-per-scope + sparse-gateway profile is a **partial
  realization** (claims 1–2 + fixed-assignment claim 3), and that work proceeds by
  strengthening the CI first, then generalizing assignment into mobility.
- Conformance clause: the CI is machine-checkable on the multi-node topology and
  no authority/cache/fanout/routing change is conformant unless that gate is green.

## A1 — Multi-node coherence gate (test + named script)

- `npm run gate:authority` → the cross-shard `cf-local-walkthrough` (4 shards,
  real DO shapes, two cross-actor sessions). This is the named inner-loop gate
  VTN0 references; required for any authority/cache/fanout/routing change.
- Added the **CI assertion**: after the walkthrough, scan captured
  `woo.commit_rejected.errors` diagnostics (CommitScopeDO logs every rejection
  with the diverged `<object>.<cell>`). This catches CI violations the existing
  step-level walkthrough **masks** — a turn that is rejected, retried, and then
  succeeds passes the step while a derived view silently disagreed with authority.

### A1 immediately found a real masked CI violation on the base
First run (zero-tolerance) FAILED on the current base: `the_outline:leave` by a
guest had its commit rejected with `read version mismatch the_chatroom.subscribers`
(transcript=2 actual=1) and `the_chatroom.session_subscribers`, then retried to
success — so the walkthrough was green while a CI violation occurred every run.
These are the **presence-projection cells** (`subscribers`/`session_subscribers`,
same class as `contents`) still sitting on the commit-validation path. This is
exactly the defect class **A4** (contents/presence-as-projection) retires.

### Ratchet design (so the gate is usable before A4)
The assertion is a **ratchet, not yet zero-tolerance**:
- `KNOWN_CI_DEBT_CELLS = { subscribers, session_subscribers, contents }` — the
  one known pre-A4 class — is allow-listed.
- ANY other diverged cell (a real property/lineage/location authority cell), any
  `post_state_mismatch`, or any `incomplete_transcript` fails the gate
  immediately. Those imply a *second writer*, not a stale projection, and are
  never excusable.
- The allow-list MUST only shrink. **A4's exit criterion is that it empties** —
  at which point this becomes zero-tolerance with no test change.

This stops *new* multiplication today without requiring A4 first, and it converts
A4 from an abstract "fix contents" into a concrete, gated objective: drive
`KNOWN_CI_DEBT_CELLS` to empty.

## Why A0+A1 had to land together
A0 without A1 is aspirational prose that the next fix can quietly violate (as the
base already did). A1 without A0 is a test asserting an invariant nobody declared
normative. Together: VTN0 declares the CI; `gate:authority` enforces it; the
allow-list names the exact remaining debt and ties it to A4.

## The rest of Phase A (A2–A5)

Phase A builds the substrate the mobile heap cannot exist without: authority is
the transcript stream, every other copy is a typed content-addressed
read-through, and the worst cell is off the validation path. Phase B (write-set
scopes, verifiable transfer, capability gossip, browser-as-node) rides on this
and is deferred until A is complete — building mobility on incoherent copies is
the exact multiplication trap that produced the current bugs.

Shared rules for every A-step: keep `gate:authority` green; the allow-list
`KNOWN_CI_DEBT_CELLS` may only shrink; no new mechanism on the
sparse-shard/cache path until A5; add nothing to the invariant ledger without
explicit justification.

### A2 — Transcript is the sole authority; one applier

**Do.** Make every durable materialization — the executable WooWorld view, the
projection rows, the gateway/edge mirrors — a deterministic function of the
*same* applied effect transcript. Delete the second, hand-maintained applier
(the `world.ts` write path that is kept "parallel with `applyPropWrite` in
`shadow-commit-scope.ts`" by comment). After A2 there is exactly one function
that turns an accepted transcript into state; all views call it.

**Why first among A2–A5.** Claims 3–5 (write-set scopes, gossip, transfer) all
assume state is a pure function of the transcript stream. While a second applier
mutates the executable mirror independently, "placement carries no semantic
meaning" (VTN0 claim 1) is literally false, and every mobility feature built on
top inherits the drift. This is the bedrock.

**CI / gate exit criterion.** `gate:authority` stays green. New assertion target:
zero `post_state_mismatch` and zero `incomplete_transcript` *from the apply path*
(these signatures are already never-excusable in the A1 gate). A2 does not by
itself shrink `KNOWN_CI_DEBT_CELLS` — it removes the *mechanism* that lets two
appliers disagree, which is the precondition for A3–A5 to be able to.

**Spec.** `spec/semantics/projection-cache.md` PC2 (row-body-complete apply)
becomes the *only* apply path, cross-linked from VTN0.

**Risk / kind.** Behavior-changing; the riskiest A-step because it touches the
hot commit-apply path. Land behind the gate, validate `smoke:cf-local` before and
after. Pure-internal: no protocol/message change.

### A3 — Provenance + content-address as a non-optional cell/page type

**Do.** Every cell/page carries `{ source: "authoritative" | "projection" |
"cache" | "fallback", content_hash, source_head }` as a **required**
discriminator (not the optional, decorative field it is today). The planner and
commit-validator **structurally refuse** to treat a non-`authoritative` cell as a
plan/commit read source — a compile/type-level refusal, replacing the scattered
runtime `isMcpGatewayShardHost(...)` checks and prose comments. This is where the
merged `preplan-authority` insight ("a sparse room stub is a routing hint, not an
authority row") is **decomposed and relanded as a type** rather than a sixth
pre-step.

**Why after A2, before A4–A5.** It is the invariant A4 and A5 must preserve.
Encoding hint-vs-authority as a type means contents-as-projection (A4) and the
read-through collapse (A5) get compile-time enforcement instead of new runtime
guards — which would themselves be fresh multiplication. In a mobile heap, a node
receives transfers and ads from peers that may be stale or untrusted; "is this
row authority or a hint?" must be a checked type on the receiver, not a
host-identity convention.

**CI / gate exit criterion.** `gate:authority` green. The A1 gate's
"any non-allow-listed diverged cell fails" rule is *strengthened*: with
provenance typed, a cache/projection row reaching a validation read is a
compile-time error, so that path cannot regress at runtime. Decompose
`preplan-authority`'s repair pre-step here: its repair need shrinks to "fetch the
authoritative cell," expressed through the type.

**Spec.** `spec/protocol/cell-authority.md` CA11 provenance promoted from
"SHOULD carry" to "MUST carry + MUST refuse non-authoritative as write source."

**Risk / kind.** Additive type + enforcement; net-removes scattered guards.
Medium risk (touches every cell/page constructor) but mechanically uniform.

### A4 — contents-as-projection, location-as-truth (CA2/CA3, the bug-densest cell)

**Do.** `live:location:<obj>` becomes the single authoritative write a movement
validates. `contents(room)`, `subscribers`, and `session_subscribers` become
**derived per-member projections that never appear on the commit-validation
path** — assembled from per-member rows, marked `source:"projection"` (enforced
by A3's type). Retire the single mutable ordered `contents`/presence cells as
authority.

**Why this is the headline correctness step.** These cells have a
storage/validation impedance mismatch — stored as an ordered list, version-hashed
as a set, read in iteration order — so value-comparison is order-sensitive while
the canonical form is not. Every movement and presence turn touches them. They
are the entire `read_version_mismatch` family: the `the_chatroom.subscribers`
violation A1 caught on the base, the `the_chatroom.contents` regression that
blocked `preplan-authority`, and `contents_cache_drift` /
`contents_order_value_mismatch` / `live_snapshot_rebase` from prior sessions. The
mobile heap's containment model *requires* this anyway: a room's contents is "a
projection assembled from multiple actor shards," not one authoritative mutable
cell.

**CI / gate exit criterion — the concrete one.** **A4 is done when
`KNOWN_CI_DEBT_CELLS` is empty** and `gate:authority` is green with
zero-tolerance (the allow-list deletes `subscribers`, `session_subscribers`,
`contents`). That is A4's definition of done — no separate metric needed.

**Why before A5.** Collapsing copies (A5) around a `contents` cell that is still
authority would bake the flawed representation into the unified store. Fix the
cell's *role* first.

**Spec.** `cell-authority.md` CA2 (`live:contents` retired as authority) and CA3
(single-location movement write) move from draft to implemented.

**Risk / kind.** Behavior-changing; do not attempt without the A1 gate. This is
where `preplan-authority` is finally decomposed away — its regression is
*dissolved*, not patched, because contents leaves the validation path entirely.

### A5 — Every non-authority view becomes a content-addressed read-through

**Do.** With one applier (A2), typed provenance (A3), and presence/contents off
validation (A4), collapse the *accidental* copies: delete `authorityCheckpoints`
(a RAM cache of the projection cache with a second applier — already missed its
prod budget); fold the REST-relay serialized blob into the single executable
view; drop per-seq snapshot blobs in favor of content-addressed pages + frame
tail. The end state is **not** "one copy" — it is many derived caches, each a
content-addressed read-through keyed on `source_head`, with exactly one authority
per cell. That is precisely the substrate a node holding any subset of cells (the
mobile heap) needs.

**Why last in A.** With A2–A4 done this is mostly *deletion*, not reconciliation.
Done earlier it would be a dangerous reconciliation of copies that still disagree.

**CI / gate exit criterion.** `gate:authority` green throughout; the per-shard
materialization count for a touched scope drops to one-per-plane (assert in the
gate that no scope is reconstructed more than once per warm turn — the
`warm_turn_refresh`/budget signal from the step-2 measurement goes to zero).

**Spec.** `projection-cache.md` PC1 ("derived rows … without creating a second
authority model") becomes structurally true; `authorityCheckpoints` —which
violates PC1— is gone.

**Risk / kind.** Behavior-changing but net-deletion. Removes the entire
"N copies under N freshness rules" surface behind the step-2 budget miss.

### End of Phase A

Authority = the transcript stream; state is a pure derivation of it; every copy
is provenance-typed and content-addressed; the bug-densest cells are out of
validation; the accidental copies are gone. This is the mobile heap's substrate —
and, not coincidentally, a correct, comprehensible system on its own.

## Phase B — the mobile-heap machinery (each a dependent step before completion)

Phase B turns the fixed-assignment substrate into the mobile heap. Each B-step is
one of VTN0's distinguishing claims (3–5). They are *dependent* steps, not
optional add-ons: the target is not reached until all of B6–B10 land. They are
deferred to Phase B because each requires the clean substrate — building any of
them on Phase-A-incomplete state recreates the multiplication. The dependency
chain is strict and is why the order is fixed.

Shared rules continue: `gate:authority` green throughout (now zero-tolerance, the
allow-list having emptied at A4); no step may reintroduce a second write path or a
non-`authoritative` validation read; new ledger entries need justification.

### B6 — Commit scope chosen by the turn's write set (VTN0 claim 3)

**Do.** Generalize from "single owner DO per scope + CA3 single-location" to
write-set-derived scope selection: actor-private state, a quiet room collocated
with one actor, a contested room with its own executor, a service/catalog scope,
or a *minted* combined/temporary scope when cells become hot — with epoch fencing
and explicit rules for multi-scope turns (acquire a combined scope, route to a
contested executor, or rewrite as multiple async turns).

**Depends on.** A2 (one applier — a multi-scope commit must apply once,
deterministically), A3 (provenance — the scope must know which cells it is
authority for), A4 (`live:location` as the single movement authority — without it,
"the write set chooses the scope" is undefined for the most common multi-object
turn). First B-step because the others route *to* scopes; the scope-selection
rule must exist before transfer and gossip can target it.

**CI / gate.** `gate:authority` green with a new multi-scope movement case added
to the walkthrough (two actors moving through the same destination concurrently):
zero `read_version_mismatch`, zero lost destination membership. Epoch-fence
rejection is a clean retryable conflict, never a silent accept.

**Spec.** VTN8 commit plane + `cell-authority.md` generalized beyond CA3.
**Risk.** Behavior-changing; concurrency-sensitive. **Closes.**
home-host-as-atomicity-boundary rigidity.

### B7 — State transfer as first-class verifiable cache-fill (VTN0 claim 5)

**Do.** Make content-addressed projection/delta/closure/snapshot-page transfer the
real post-execution path: every remote turn reply carries the transcript plus
receiver-authorization-filtered, content-hash + receipt verifiable state transfer
that warms the caller. Resolve the four transfer modes (retire the dead `closure`
diagnostic mode or realize it as a real bounded mode). State transfer never grants
write authority — it installs `source:"cache"` rows (A3 type) only.

**Depends on.** A3 (content-address + provenance are the transfer's verification
and install discipline), A5 (the read-through substrate is what a transfer fills),
B6 (a transfer follows a remote *commit*, which now lands in a write-set-chosen
scope). Before B8 because gossip routing is only useful if a miss is cheaply
warmed — B8's `factor` ranking includes `estimated_transfer_cost`, which presumes
B7's transfer exists and is measurable.

**CI / gate.** A two-node gate case: actor executes remotely, receives transfer,
the *next* same-object turn is local (no second remote fetch). Assert the
installed rows are `source:"cache"` and never satisfy a commit-validation read
(A3 enforcement). No CI violation across the warm-up.

**Spec.** VTN12 state plane; the four transfer modes pinned. **Risk.**
Behavior-changing. **Closes.** per-miss cold remote re-reads; "warm the caller"
becomes structural rather than a checkpoint hack.

### B8 — Capability gossip routing (VTN0 claim 4) — the keystone

**Do.** `TurnKey` extraction from command planning (typed read/write atom hashes +
effect mask); `ExecCapabilityAd` (`covers`/`accepts` Bloom + opaque `factor` +
scope/epoch/head + TTL); multi-node candidate ranking
(`latency + factor + transfer_cost + failure_penalty`); refuse-cheaply on a false
positive. The single-node `capability-ad` / `turn-key` / `shadow-turn-network`
scaffolding becomes the real routing layer; the **static route table is retired
behind it**. Ads route; commit validation still proves authority.

**Depends on EVERYTHING above** — which is why it is last among structural work:
A2 (clean authority to advertise a true head), A3 (provenance, so an ad's
`covers` claim is about authority not stale cache), A4 (stable movement authority
to key scope ads on), A5 (read-through, so a node can honestly advertise what it
can cheaply serve), B6 (ads are *per commit scope*), B7 (ranking needs transfer
cost). Build gossip before any of these and every false-positive route lands on
incoherent state.

**CI / gate.** A ≥3-node gate case where execution *migrates*: a cold miss warms
an actor node, the next turn routes local, a contended object routes to its
busy-room executor. Assert: zero CI violation under routing, false-positive
refusals are bounded and never corrupt state, no global enumeration on the hot
path (big-world discipline).

**Spec.** VTN11 capability gossip, promoted from draft/not-on-prod-path to
implemented. **Risk.** Highest-reach behavior change. **Closes.** the
location-oracle on the hot path; enables discovered/mobile placement — the
target's distinguishing bet.

### B9 — Browser/edge as a real narrow-authority node (Execution + Live planes)

**Do.** Fold the divergent browser holder into the same node model: a node on the
Execution and Live planes that may execute turns for its logged-in actor and
subscribed surfaces, may **not** commit without server-side scope validation,
receives only authorized transfers (A3 filter), and advertises nothing — or one
narrow session-local ad to its relay. Browser execution becomes a latency
optimization, not a trust assumption.

**Depends on.** B6 (browser commits land in a validated write-set scope), B7
(browser cache is filled by authorized transfer), B8 (the narrow ad is a
restricted case of the gossip layer). Last functional step because it is the
most-constrained node and must inherit, not re-implement, the general machinery.

**CI / gate.** Extend the gate (or the browser integration suite) so a
browser-node turn drives chat/pinboard/dubspace through server-side commit
validation with zero CI violation; optimistic local execution reconciles cleanly
on conflict (patch-forward, no divergent holder state).

**Spec.** `spec/protocol/browser-host.md` re-folded into VTN as a node profile,
not a separate protocol. **Risk.** Behavior-changing; client-coupled. **Closes.**
the "browser is a divergent holder protocol" fork.

### B10 — Retire the compensating mechanisms (pure deletion; completion)

**Do.** Only once B6–B9 hold, delete the scaffolding that existed to compensate
for the missing structure: the `preplan-authority` pre-step (its insight is A3's
type, its repair need gone once contents left validation at A4); the
checkpoint → catch-up → repair → seed ladder (subsumed by A5 read-through + B7
transfer); the static route table (subsumed by B8 gossip); any single-node-only
simulator paths that B7/B8 replaced.

**Depends on.** All of A and B6–B9 — each deletion is safe only because its
replacement is in place. This is the completion step: when B10 is done, no
mechanism remains whose only reason to exist was to paper over a missing piece of
the model.

**CI / gate.** `gate:authority` green with the compensations gone; net line count
down; the invariant ledger at its minimum. **Risk.** Pure deletion. **Closes.**
the compensating-mechanism layer entirely — the thing whose accumulation made the
work hard in the first place.

### Completion

The target is reached when B10 lands: deterministic VM turns over a mobile,
actor-local heap; authority is the transcript stream; many provenance-typed
content-addressed copies, exactly one authority per cell; commit scope chosen by
write set; remote execution warms the caller by verifiable transfer; routing is
capability gossip with no location oracle; the browser is a narrow node; and none
of the compensating mechanisms remain. Every step from A2 to B10 either removed an
invariant or rode on one already removed — which is the whole point of the
ordering: complexity never multiplied, because no mobility machinery was ever
built on copies that could disagree.
